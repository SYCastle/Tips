from matplotlib.colors import ListedColormap

def plot_decision_boundary(clf, X, y, axes=[0, 7.5, 0, 3], iris=True, legend=False, plot_training=True):
    x1s = np.linspace(axes[0], axes[1], 100)
    x2s = np.linspace(axes[2], axes[3], 100)
    x1, x2 = np.meshgrid(x1s, x2s)
    X_new = np.c_[x1.ravel(), x2.ravel()]
    y_pred = clf.predict(X_new).reshape(x1.shape)
    custom_cmap = ListedColormap(['#fafab0','#9898ff','#a0faa0'])
    plt.contourf(x1, x2, y_pred, alpha=0.3, cmap=custom_cmap)
    if not iris:
        custom_cmap2 = ListedColormap(['#7d7d58','#4c4c7f','#507d50'])
        plt.contour(x1, x2, y_pred, cmap=custom_cmap2, alpha=0.8)
    if plot_training:
        plt.plot(X[:, 0][y==0], X[:, 1][y==0], "yo", label="Iris-Setosa")
        plt.plot(X[:, 0][y==1], X[:, 1][y==1], "bs", label="Iris-Versicolor")
        plt.plot(X[:, 0][y==2], X[:, 1][y==2], "g^", label="Iris-Virginica")
        plt.axis(axes)
    if iris:
        plt.xlabel("Petal length", fontsize=14)
        plt.ylabel("Petal width", fontsize=14)
    else:
        plt.xlabel(r"$x_1$", fontsize=18)
        plt.ylabel(r"$x_2$", fontsize=18, rotation=0)
    if legend:
        plt.legend(loc="lower right", fontsize=14)

6.4 CART訓練アルゴリズム¶

scikit-leanはCART(Classification ANd Regression Tree)アルゴリズムを使って決定木を訓練する。

1.一つのフューチャー(特徴量)kとしきい値tk(例えば「花弁の長さ」<「2.45」cm)を使って、訓練セットを二つのサブセットに分割。

2.(アルゴリズムはどうやってkとtkを選ぶか?)コスト関数(左右のサブセットの不純度と左右のサブセットのインスタンス数の和)が最小になるようなペアを探して学習する。

3.訓練セットの2分割に成功したら、サブセットを同じ理論で分割し...繰り返し...再帰的に実行していく。

4.深さの制限に達するか、不純度を下げる分割が見つからなければ再帰を中止。

残念ながら、最適な木を見つけるという問題は「NP完全問題」であり、計算量がO(exp(m)なので、ごく小規模な訓練セットでも、手に負えなくなる可能性があるので、「まずまずの良い解」で満足しなければならない。

6.5 計算量¶

スキップ(scikit-lean内で多少高速化する仕組みはあるが、訓練の計算量は、O(n x mlog(m)))と、予測の計算量O(log2(m))より多いっていう話)

6.6 ジニ不純度かエントロピーか¶

デフォルトではジニ係数が使われるが、エントロピー(熱力学で分子の乱雑さの指標として使われる)へ切り替え可能。 (セットが一つのクラスに属するインスタンスだけから構成される場合、そのエントロピーは「0」)

どちらを使うかだが、どちらを使っても同じような木になるが、ジニ不純度の方がわずかではあるが高速である。 また、両者の違いとして、ジニ不純度は最頻出クラスを木の中に専用ブランチという形で分離する傾向があるのに対し、エントロピーはそれよりもわずかに平衡の取れた木を作る傾向がある。

6.7 正規化ハイパーパラメータ¶

決定木は訓練データに対してほとんど先入観を持たない(例えば、線形モデルはデータが線形に分布していると最初から決めつけている)。 なので、訓練データに合わせて自ら調整し、訓練データに密接に適合するが、過学習しやすい。

このようなモデルはノンパラメトリックモデルと呼ばれることが多い。 (パラメータの数が勝っていしていない)

線形モデルなどのパラメトリックモデルはあらかじめ決められてた数のパラメータがあるため、自由度が制限され、過学習のリスクが低くなっている。

過学習を防ぐには、決定木の訓練中、制限をかける必要がある。

例えば、決定木の深さ上限の制限の他に、ノードを分割するために必要なサンプリング数の下限値等...

正規化なしの場合の決定木と、min_samples_lea=4というハイパーパラメータで正規化した場合を比較してみよう

6.9 不安定性¶

これまでの説明で、決定木は「解釈しやすく、柔軟で使いやすく(回帰や分類ができるかから)」といったメリットが挙げられたが、デメリットもいくつかある。

「決定境界は直交する」

訓練セットの回転によって、結果(決定木)が大きく変わる。 例えば、1つの境界で引けたデータセットを45度回転させて場合、階段のように複数の境界線で分類するような決定木に代わる(=リージョンが増えるので、汎化性能が落ちる)。 この問題は、訓練データをよりよい向きに変えられることが多い主成分分析(次元削減にも効果あり)を使えば、ある程度軽減できる。

Sensitivity to training set details¶

より一般的に言うと、決定木の最大の問題は訓練セットの小さな変化に敏感すぎることである。 例えば、irisの訓練セットから花弁の幅が最も長いインスタンスを取り除いて決定木を訓練すると、以前の決定木とは大きく異なる以下のようなモデルが得られる。
